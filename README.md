Enhanced-Cpp-ThreadPool
一个工业级、高可靠的 C++ 动态弹性线程池。不仅实现了基础的任务调度，更引入了动态扩缩容、背压保护、异常隔离以及实时指标监控等高级特性，适用于高并发服务器后端及密集型计算场景。

核心特性
动态弹性伸缩：采用“正式工+临时工”模型。根据任务负载自动创建临时线程，并在闲置超时后自动回收，完美平衡响应速度与系统资源消耗。

背压保护 (Backpressure)：内置有界任务队列，防止因生产过快导致内存溢出 (OOM)，确保系统在高压环境下的稳健性。

异常安全屏障：任务执行过程受 try-catch 保护。单一任务的崩溃（异常抛出）不会导致工人线程退出或池子瘫痪。

实时指标度量：基于 std::atomic 实现无锁监控，支持实时获取：当前线程总数、忙碌线程数、排队任务数及累计完成总数。

泛型任务支持：利用模板元编程与 std::future，支持提交任意签名的函数或 Lambda 表达式，并能方便地获取异步执行结果。

快速开始
1. 引入头文件
本项目为 Header-only 设计，只需将 ThreadPool.hpp 放入你的工程即可。

2. 基础使用
C++
#include "ThreadPool.hpp"
#include <iostream>

int main() {
    // 初始化：核心 2 线程，最大 5 线程，队列容量 100
    ThreadPool pool(2, 5, 60, 100);

    // 提交任务并获取结果
    auto future = pool.submit(1, [](int a, int b) {
        return a + b;
    }, 10, 20);

    std::cout << "结果: " << future.get() << std::endl;
    return 0;
}
核心逻辑说明
动态扩缩容算法
池子维护 core_threads（核心）和 max_threads（最大）两个阈值。

当 空闲线程 == 0 且 总线程 < 最大线程 时，触发临时线程创建。

临时线程在 wait_for 等待任务超时后，会自动触发自裁逻辑，实现资源回笼。

异常处理流
所有提交的任务都被包装在执行屏障中：

C++
try {
    task_func(); // 执行业务逻辑
} catch (...) {
    // 捕获异常，防止工人线程死掉，并记录错误
}
性能与监控
通过内置的原子计数器，你可以轻松对接 Prometheus 或 Grafana 等监控系统，实现线程池状态的可视化：

get_cur_threads(): 实时掌控扩容状态。

get_running_tasks(): 监测当前系统繁忙程度。
